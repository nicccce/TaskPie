# 任务派（TaskPie）产品报告
## 1. 产品概述
任务派是一款面向高校学生小组协作的任务管理平台。它旨在解决课程项目、科研小组、社团活动等场景中常见的任务分配不清、进度难以跟踪、成果分散、协作效率低等问题。通过任务派，团队成员可以清晰地了解各自的任务、实时更新进度、集中提交成果，组长则可以有效统筹全局、审批成果并掌握团队动态。系统支持公开小组搜索与申请加入，组长可邀请成员，所有成员仅能查看自己所在小组的详细信息，保障数据隔离。

## 2. 系统角色与权限

| 角色 | 权限说明 |
| --- | --- |
| 管理员 | 系统级管理，负责用户管理、权限分配、日志监控，可查看所有小组数据。 |
| 组长 | 团队级管理，可创建小组、设置小组可见性、邀请成员、审批加入申请、发布任务、分配任务、审批成果、查看统计图表。 |
| 组员 | 执行任务，可查看被分配的任务、提交成果、查看个人任务进度和反馈。|

每个用户可同时属于多个小组，在不同小组中可拥有不同角色。


## 3. 核心功能模块
### 3.1 用户管理
| 模块 | 说明 |
| --- | --- |
| 注册/登录 | 用户通过学号、姓名、密码注册，登录后获得JWT令牌。 |
| 个人信息维护 | 修改姓名、密码、联系方式等。 |
| 用户搜索 | 组长可通过学号或姓名搜索用户，用于发送邀请。 |

### 3.2 小组管理
| 模块 | 说明 |
| --- | --- |
| 创建小组 | 用户可创建小组，成为组长，并设置小组可见性（公开/私有）。 |
| 小组搜索 | 所有用户可根据名称、描述等关键词搜索公开小组，查看公开信息（名称、简介、成员数等）。 |
| 加入申请 | 用户可向公开小组发起加入申请，组长在申请列表中查看并处理（同意/拒绝）。 |
| 邀请成员 | 组长可通过搜索用户，向用户发送邀请，用户同意后加入小组。 |
| 成员管理 | 组长可查看成员列表、修改成员角色（组长/组员）、移除成员；成员可主动退出小组。 |
| 小组信息维护 | 组长可修改小组名称、简介、可见性。 |

### 3.3 任务管理
| 模块 | 说明 |
| --- | --- |
| 发布任务 | 组长可创建任务，指定任务标题、详细描述、截止日期、优先级，并可选择分配给具体组员。 |
| 任务看板 | 以卡片形式展示任务，按状态（待处理、进行中、待审批、已完成、已驳回）分组，支持拖拽调整状态。 |
| 任务详情 | 查看任务完整信息，包括描述、附件、截止时间、当前状态、所有提交记录和审批反馈。 |
| 任务评论 | 成员可在任务下方评论、@其他成员，并上传附件，所有评论形成动态流。 |
| 子任务与依赖（扩展） | 任务可拆分为子任务，设置前置依赖。 |

### 3.4 成果提交与审批
| 模块 | 说明 |

| 提交成果 | 组员完成任务后，可在任务详情页提交成果，支持文本描述和附件上传。 |
| 审批流程 | 组长收到提交通知后，进入审批界面查看成果内容，可给出反馈意见并选择“通过”或“驳回”。 |
| 多级审批（扩展） | 可配置多级审批。 |

### 3.5 数据统计与可视化
| 模块 | 说明 |
| --- | --- |

| 成员贡献度 | 柱状图展示每位成员完成的任务数量。 |
| 任务状态分布 | 饼图展示任务状态比例。 |

| 个人任务统计 | 每位成员可查看自己的任务完成情况、按时提交率。 |

### 3.6 消息通知
| 模块 | 说明 |
| --- | --- |
| 实时通知 | 任务分配、审批结果、评论@、申请结果等事件触发通知，用户在登录后弹窗提醒。 |

## 4. 核心业务流程
### 4.1 小组加入流程
用户搜索小组：用户在系统内搜索公开小组，查看小组公开信息。

发起申请：用户点击“申请加入”，向组长发送申请。

组长审批：组长在“申请列表”中看到待处理的申请，可选择“同意”或“拒绝”，并填写反馈（可选）。

结果通知：用户收到申请结果通知，若同意则自动成为小组成员。

邀请流程：

组长搜索用户：组长通过学号或姓名搜索用户。

发送邀请：组长向目标用户发送邀请。

用户确认：用户收到邀请通知，可选择“接受”或“拒绝”，接受后自动加入小组。

### 4.2 任务生命周期流程
创建任务（组长）→ 2. 分配任务（组长或组员认领）→ 3. 执行任务（组员）→ 4. 提交成果（组员）→ 5. 审批成果（组长）→ 6. 任务结束（通过）或 驳回修改（组长驳回后返回步骤3）

### 4.3 权限隔离逻辑
小组公开信息：名称、简介、成员数量、创建时间等（对全体用户可见）。

小组私有信息：任务详情、成员详细信息（如姓名、学号）、提交记录等（仅小组成员可见）。

数据访问控制：所有接口校验当前用户是否为小组内成员，非成员无法访问私有数据。

## 5. 管理核心逻辑
### 5.1 状态管理
任务状态：待处理 → 进行中 → 待审批 → 已完成 / 已驳回（驳回后回到进行中）

提交状态：已提交 → 已通过 / 已驳回

申请状态：待处理 → 已同意 / 已拒绝

邀请状态：待确认 → 已接受 / 已拒绝
